What is Web Accessibility?

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. More specifically, people can perceive, understand, navigate, and interact with the Web, can contribute to the Web. Web accessibility encompasses all disabilities that affect access to the Web, including auditory, cognitive, neurological, physical, speech, visual. Web Accessibility

How Web Accessibility Started?

Historicity of Web Accessibility

In 1995, Dr. Cynthia Waddell published a web design accessibility standard for the City of San Jose’s Office of Equality Assurance. In September of 1996, Berners-Lee circulated a newsletter to W3C members asking for guidance on how to handle accessibility in the face of the web’s growing complexity. In April of 1997, the WAI(Web Accessibility Initiative ) officially launched. In December of 1997, HTML 4.0 was released to the world, packed with new features and tags. In May of 1999, the WCAG(Web Content Accessibility Guidelines) were published as an official W3C recommendation.

Barriers

People are different and websites should be designed for universal accessibility. People with disabilities require barriers to be removed and alternate options to be provide in order to navigate and use websites resources.Such disabilities include:

  • Visual impairements: Cannot see graphics
  • Hearing Impairement: cannot hear audio
  • Physical impairement: cannot use mouse
  • Learning Disabilities: difficulty to navigate poorly organized sites
  • Low english proficiency
  • Slow internet connection
What is Web Accessibility Initiative?

Web Accessibility initiative

The Web Accessibility Initiative (WAI) is an initiative developed by the World Wide Web Consortium to help make the internet more accessible to people with disabilities. The WAI applies to all types of disabled populations, including the growing elderly population.
The W3C Web Accessibility Initiative (WAI) develops standards and support materials to help you understand and implement accessibility.

Principles to follow

Web Accessibility principles

  • Providing text alternatives to visual content
  • providing text alternatives to audio content
  • Avoiding use of color to convey essential information
  • Using high contrast foreground and background colors
  • Using relative instaed of absolute units
  • providing clear, simple design
  • Clarifying language usage

Web Accessibility Components

Different components of web development and interaction work together in order for the web to be accessible to people with disabilities.
